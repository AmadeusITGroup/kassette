      • OptionalcaseSensitive?: boolean

        Configures whether keys are case sensitive (i.e. whether keys differing only with the case, such as a and A, are considered different). If keys are not case sensitive, their case will be changed to lower case before being hashed or being compared with the provided whitelist or blacklist. By default, keys are considered case sensitive for the query and case insensitive for the headers.

          Configures whether the list of keys is a whitelist (specifying explicitly all the items to include, any other item being excluded) or a blacklist (specifying explicitly all the items to exclude, any other item being included). Defaults to whitelist.

    Remarks

    This can be an object with the following properties:

    • include: true or false as defined in IncludableSpec

    • filter: a function as defined in FilterableSpec. Note that if filter is provided, the other options below are ignored.

    • caseSensitive: whether keys should be treated case sensitive or not. true by default. When set to false, the output object contains lower cased keys.

    • keys: a list of keys to keep if in whitelist mode or to reject if in blacklist mode. If caseSensitive is false, comparison of keys is not case sensitive. If keys is not specified, all keys are included by default.

    • mode: whitelist (default) or blacklist
